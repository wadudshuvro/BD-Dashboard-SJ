# Unused Analysis

## Frontend Modules
- `src/components/documentation/TableOfContents.tsx` – Table-of-contents renderer created for earlier documentation experiments; no current imports in the codebase.
- `src/components/documentation/MarkdownRenderer.tsx` – Custom markdown component superseded by the new admin documentation page.
- `src/data/mockData.ts` (Admin panel mocks) – Contains legacy seed data for dashboards; production flows now load from Supabase.

## Edge Functions & API
- `supabase/functions/openai-test` – Intended for credential smoke tests; should remain disabled in production deployments or moved behind admin-only routing.
- `supabase/functions/import-hours` – Ready for Toggl/Harvest import but not wired into the UI yet; invocation scripts live only in docs.

## Database Entities
- `brand_analytics_data` – Populated exclusively via `n8n-analytics-manage`; the React UI has not yet surfaced visualization components for this dataset.
- `activecollab_task_data` – Mirror table for ActiveCollab payloads. Currently consumed by edge functions but unused in client dashboards.
- `code_analysis_results.severity` – Column populated by `analyze-codebase`; no UI currently filters by severity, so values remain informational only.
- `team_daily_summaries.productivity_score` – Generated by AI summaries but not displayed; candidate metric for upcoming dashboards.

## Recommendations
1. **Confirm Ownership:** Decide whether `TableOfContents` and `MarkdownRenderer` should be deleted or revived for rich docs navigation.
2. **Expose Analytics:** Build frontend cards to surface `brand_analytics_data` and `activecollab_task_data` (e.g., traffic charts, task sync health).
3. **Leverage Scoring Fields:** Highlight `code_analysis_results.severity` and `team_daily_summaries.productivity_score` inside admin review screens so the stored metrics drive action.
